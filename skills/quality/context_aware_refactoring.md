# Skill: Context-Aware Refactoring
## 概要
AIエージェントと人間が文脈を誤解しないよう、1つの関数を30行以内に凝縮し、可読性と保守性を最大化する。

## 実行プロセス
1. **関数の単一責任（SRP）の追求**:
   - 1つの関数が担う役割を1つに絞り、条件分岐やループが複雑化している場合は機械的にサブ関数へ分割する。
2. **AI親和性の向上（Self-Documenting Code）**:
   - コメントに頼らず、命名（変数・関数名）と構造だけで意図が伝わるようにリファクタリングする。
   - これにより、AIがコードを読み取る際のハルシネーション（誤解）を未然に防ぐ。
3. **不要な抽象化の排除**:
   - 複雑すぎるデザインパターンの過剰適用を避け、最もシンプルで直接的な実装を選択する。

## 成功基準
- 各関数が30行以内に収まり、初見のAIエージェントがドキュメントなしで正しくロジックを理解できること。

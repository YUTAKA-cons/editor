# Skill: Code Redundancy & Complexity Audit

## 役割
AIが生成したコードに「重複」や「過剰な肥大化」がないかを厳格に検査する。

## 実行プロセス
1. **重複コード検知**: 異なるクラス間で類似したロジックが書かれていないかスキャンし、PLに「共通化」を命じる。
2. **インスタンス化コストの監視**: 無駄な `new`（インスタンス生成）が繰り返されていないか確認し、SingletonやStaticへの置き換えを提案する。
3. **データ構造の妥当性評価**: 順序が不要な場所でListが使われていないか（Setで十分ではないか）等を指摘し、計算量を最適化させる。

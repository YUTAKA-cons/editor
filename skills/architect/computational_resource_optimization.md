# Skill: Computational Resource Optimization
## 概要
メモリ消費、CPU負荷、およびコードの肥大化を最小化するための構造的最適化を行う。

## 実行プロセス
1. **データ構造の最適化（計算量 $O(n)$ から $O(1)$ への転換）**:
   - 言語を問わず、探索が必要な箇所で「リストのループ（線形探索）」を避け、ハッシュセットやマップ（平均定数時間）を優先的に使用する。
   - 重複排除をロジックで書かず、言語標準のSet型（HashSet, Set等）に委ねてコード行数を削減する。
2. **軽量データ型の選択**:
   - 言語がサポートする場合、参照型（オブジェクト）よりも値型（プリミティブ、構造体）を優先し、ヒープメモリの消費とガベージコレクションの負荷を抑える。
   - オートボクシングや不要な型変換を排除する。
3. **初期容量の明示**:
   - 動的配列やハッシュテーブルを生成する際、要素数が予測可能であれば初期容量を指定し、内部の再確保（Resize）コストを最小化する。

## 成功基準
- 同等の機能が、リファクタリング前よりも少ないメモリ使用量とコード行数で実現されていること。
